# Redshift Concepts And Trial Notes

Redshift is included conceptually in this phase, but the student should not do hands-on Redshift labs yet.

## Why Cover Redshift Now

The student should understand the warehouse concepts that appear across platforms:

- compute resources
- storage
- databases
- schemas
- permissions
- query history
- cost controls
- workload monitoring

## Redshift Serverless Terms

- Namespace: collection of database objects and users.
- Workgroup: compute resources used to run queries.
- RPU: Redshift Processing Unit, used for Serverless compute capacity.
- Query editor: browser-based SQL workspace.

## Trial Notes

Amazon currently advertises an Amazon Redshift Serverless free trial for eligible accounts that have not used Redshift Serverless before. The official AWS page describes a trial credit that can be used within a limited trial period.

Do not start hands-on Redshift workloads for this course phase unless you explicitly decide to add a Redshift lab later.

## Cost Safety Habits

- Confirm eligibility before creating resources.
- Use Serverless trial resources only if needed.
- Monitor usage and billing dashboards.
- Delete unused workgroups or namespaces after labs.
- Prefer Snowflake for this phase's hands-on cloud work.
